# Vendor Performance & Retail Analytics Dashboard

## 📌 Project Overview
This project delivers an end-to-end data analytics pipeline designed to monitor and evaluate vendor performance, sales distribution, and financial health. By joining and modeling complex transactional datasets, the dashboard provides actionable insights into sales efficiency, purchase distribution, and profit maximization.

---

## 🛠️ Tech Stack & Architecture
* **Data Processing & Engineering:** Python (Pandas) / SQL – Used for extracting raw transactional data, managing joins, and calculating core engineering metrics.
* **Data Modeling & BI:** Power BI – Utilized for structural star-schema modeling, DAX measurements, and interactive data visualization.

## 💡 Key Business Insights
* **Revenue vs. Investment:** Achieved total sales of **$441.41M** against total purchases of **$307.34M**, yielding a robust healthy baseline.
* **Profitability Margin:** Maintained an overall business **Profit Margin of 38.72%**, showcasing high pricing efficiency across primary brands.
* **Vendor Concentration:** A significant portion of the total purchase distribution is heavily concentrated among the top 10 vendors (e.g., Diageo North America Inc, Martignetti Companies), highlighting key strategic partners.
* **Capital Risk:** Identified **$2.71M in Unsold Capital**, marking key operational areas for inventory optimization and stock turnover improvement.

## ⚙️ How to Run This Project
1. **Data Pipeline:** Review the Jupyter Notebooks/SQL scripts in the repository to see the raw data transformation and feature engineering steps.
2. **Power BI Dashboard:** Download the `.pbix` file from this repository and open it in *Power BI Desktop* to explore interactive slicing (by Brand, Vendor, or Sales tiers).
